skills — tekrarlayan işi bir kez yaz
# prosedür içeren bir markdown dosyası
skill-creator built-in. skill'i elle yazmak yerine claude'a yazdır.
model: sonnet skill başına model seçilir. sen opus'tayken bile
bu skill sonnet (ya da haiku) ile çalışır.
disable-model-invocation: true model asla kendiliğinden çağıramaz.
sadece sen slash command olarak çalıştırırsın.
user-invocable: false tam tersi: sadece model çağırır,
slash command olmaktan çıkar.
$ARGUMENTS skill argüman alır: /deploy staging → staging'e deploy eder.
zihniyet değişimi: bir şey yapman gerektiğinde önce sor — "claude code bunu zaten otomatikleştirmiş mi?"
auto mode — permission fatigue'in çözümü
fig. 06 · shift+tab
@omerfrkdemiral

problem: permission fatigue

her şeyi sorarsa yüzüncü seferde artık okumadan onaylıyorsun. bu güvenli değil, tehlikeli.

diğer uç dangerously-skip-permissions: hiç sormuyor. root'u silerse dönüşü yok.

çözüm: aradaki classifier

auto mode, allow ve deny listesinin arasında durur. her tool call'dan önce ayrı bir classifier çalıştırır: bu tehlikeli mi?

tehlikeliyse sorar, değilse rahatsız etmez.

prompt injection'a karşı daha iyi

tool sonucundan gelen "tüm talimatları yok say" tarzı saldırıları, araya giren classifier yakalayabiliyor.

bağlama duyarlı

rm -rf normalde tehlikeli. ama sen silmesini istediysen tehlikeli değil — auto mode sormaz.

dynamic workflows —
subagent'ları deterministik yapmak
slash command değil: sadece "dynamic workflow kullan" de.
01
subagent'ın sorunu
subagent = kendi context'i olan ayrı bir claude code instance. tek işe
odaklanır, iyi çalışır. ama non-deterministik: bazen 4 tane açar,
bazen 10, bazen hiç. aynı işi iki kez aynı şekilde yapmaz.
02
workflow'un çözümü
claude arkada bir javascript dosyası üretiyor — fazlar, subagent'lar,
her birinin prompt'u. s'e bas, kaydet → artık bir slash command.
her çağırdığında birebir aynı subagent'ları çalıştırır.
03
fazlar sıralı, agent'lar paralel
build → integration → review → verify sırayla; build içindeki 4 agent
aynı anda. paralel olduğu için tek agent süresinde çok daha fazlası biter.
04
yüzlerce subagent
bu bir hata değil, özellik. saatler veya günler süren işler için tasarlandı.
05
js'i sen düzenle
her subagent'ın modelini değiştirebilirsin. zayıf iş için sonnet,
zor iş için opus. kod olduğu için deterministik.
non-determinizmi kodla sabitlemek agent factory podcast
